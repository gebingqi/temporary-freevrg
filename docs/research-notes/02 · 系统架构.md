# 02 · 系统架构

### **2\.1 整体架构（五层）**



```Plain Text
┌─────────────────────────────────────────────────────────────────┐
│                   Layer 0: 数据基础设施                          │
│  FreeBSD-SA git repo  │  FreeBSD src git history  │  (RAG库)   │
└────────────────────────────┬────────────────────────────────────┘
                             ↓
┌─────────────────────────────────────────────────────────────────┐
│           Layer 1: Corpus 构建（纯脚本，无 LLM）                │
│   解析 SA → 提取 CVE / fix commit / 受影响文件                  │
│   → 抽取 diff + 修复前后完整函数 + 调用上下文                   │
│   → 输出结构化 JSON，按时间做 train/val/test split              │
└────────────────────────────┬────────────────────────────────────┘
                             ↓
┌─────────────────────────────────────────────────────────────────┐
│           Layer 2: 模式抽象（LLM-A · Pattern Agent）           │
│   输入：单条 corpus record                                       │
│   输出：pattern.md（自然语言 + source/sink/sanitizer 结构字段）  │
└────────────────────────────┬────────────────────────────────────┘
                             ↓
┌─────────────────────────────────────────────────────────────────┐
│           Layer 3: 规则生成（LLM-B · Rule Agent）              │
│   输入：pattern.md + few-shot CodeQL 示例                        │
│   输出：rule.ql + 元数据注释                                     │
└────────────────────────────┬────────────────────────────────────┘
                             ↓
┌─────────────────────────────────────────────────────────────────┐
│           Layer 4: 验证回路（确定性 Validator）                  │
│   ① codeql query compile   ② 召回测试（vuln版本应命中）         │
│   ③ 误报测试（fix版本不应命中）  ④ 失败 → LLM修复（最多3轮）    │
│   ⑤ 通过 → 入规则库  /  多次失败 → 人工审查队列                 │
└────────────────────────────┬────────────────────────────────────┘
                             ↓
┌─────────────────────────────────────────────────────────────────┐
│           Layer 5: 评估与应用                                    │
│   学术评估（test set P/R/F1）  │  实战扫描（FreeBSD-CURRENT）   │
│   新漏洞确认 → 负责任披露  │  回流 Corpus → 下一轮迭代           │
└─────────────────────────────────────────────────────────────────┘
```



---



### **2\.2 各层职责说明**



|层|名称|是否用 LLM|核心输入|核心输出|
|---|---|---|---|---|
|Layer 0|数据基础设施|否|FreeBSD git|原始数据|
|Layer 1|Corpus 构建|否|SA \+ src|JSON corpus|
|Layer 2|模式抽象|是（LLM\-A）|corpus record|pattern\.md|
|Layer 3|规则生成|是（LLM\-B）|pattern\.md|rule\.ql|
|Layer 4|验证回路|是（LLM\-C，修复）|rule\.ql \+ val set|验证结果|
|Layer 5|评估与应用|否（扫描阶段）|验证通过的规则|告警 \+ 指标|



---



### **2\.3 数据流说明**



```Plain Text
FreeBSD-SA (.asc)
  → 解析脚本 → corpus_record.json
    → Pattern Agent (LLM-A) → pattern.md
      → Rule Agent (LLM-B) → rule.ql
        → Validator
            ├─ 通过 → rules/ 规则库
            └─ 失败 → Rule Repair (LLM-C，最多3轮)
                       ├─ 修复成功 → rules/ 规则库
                       └─ 修复失败 → 人工审查队列
```
