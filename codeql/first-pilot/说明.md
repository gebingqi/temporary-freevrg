# 首批 QL 验证包

## 定位

本目录是基于 `RuleAgent试点包` 整理的正式第一版候选 QL 和验证包，用于推进 FreeVRG 当前最小闭环：

```text
pattern -> rule_input.json -> candidate .ql -> compile -> vulnerable recall -> fixed false-positive check
```

同事提供的 `D:\桌面\学习\实习\rules` 目录只作为 CodeQL 写法和环境参考，不作为本包的正式成果来源。

## 内容

| 路径 | 说明 |
|------|------|
| `qlpack.yml` | CodeQL C/C++ pack 定义 |
| `queries/*.ql` | 两条正式第一版候选 QL |
| `metadata/*.metadata.json` | 输入来源、hash、scope 和预期验证目标 |
| `notes/*.notes.md` | 每条规则的建模说明、限制和修复优先级 |
| `validation/验证矩阵.md` | 5 个历史实例的 vulnerable/fixed 验证矩阵 |
| `validation/验证结果模板.json` | 后续跑数据库验证时填写的结构化模板 |
| `validation/编译记录.md` | 本机 CodeQL 编译检查记录 |
| `validation/验证数据库准备记录.md` | 10 个最小 vulnerable/fixed CodeQL database 的准备与 smoke 记录 |
| `minimal-validation-databases/source/` | 5 个试点 CVE 的最小 vulnerable/fixed harness 源码 |
| `scripts/run_smoke_validation.sh` | 在本地重新生成 smoke 验证结果的脚本 |

## 当前状态

- 已整理两条候选 QL。
- 两条候选 QL 已通过本机 CodeQL 编译检查。
- 已准备 5 个试点 CVE 的 vulnerable/fixed 最小 harness 源码，可据此重新建库。
- 已保留 smoke 验证脚本和历史验证记录，但数据库、日志和 SARIF 输出不再随仓库存放。
- 当前 smoke 结果显示：数组/描述符规则能命中 3 个 vulnerable harness，但 fixed 仍报；协议长度规则在 2 个 vulnerable harness 上暂未命中。下一步应做 miss/still_reports 归因。

## 本轮不做

- 不做 FreeBSD-CURRENT 大规模扫描。
- 不处理多 CVE composite diff。
- 不把 `PASS_WEAK` pattern 纳入本包。
- 不把未验证 `.ql` 当作正式规则库。

## 仓库存放约定

为避免把可再生运行产物提交进版本库，以下路径不再保留在仓库中：

- `minimal-validation-databases/db/`
- `minimal-validation-databases/logs/`
- `minimal-validation-databases/results/`

需要时可根据 `validation/` 记录和 `scripts/run_smoke_validation.sh` 重新生成。
