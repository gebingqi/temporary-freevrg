# 首批 QL 验证包

## 定位

本目录是基于 `RuleAgent试点包` 整理的正式第一版候选 QL 和验证包，用于推进 FreeVRG 当前最小闭环：

```text
pattern -> rule_input.json -> candidate .ql -> compile -> vulnerable recall -> fixed false-positive check
```

同事提供的 `rules` 参考目录只作为 CodeQL 写法和环境参考，不作为本包的正式成果来源。

## 内容

| 路径 | 说明 |
|------|------|
| `qlpack.yml` | CodeQL C/C++ pack 定义 |
| `queries/*.ql` | 两条正式第一版候选 QL |
| `metadata/*.metadata.json` | 输入来源、hash、scope 和预期验证目标 |
| `notes/*.notes.md` | 每条规则的建模说明、限制和修复优先级 |
| `scripts/run_smoke_validation.sh` 与 `scripts/脚本使用说明.md` | Linux 上复跑最小 smoke 验证的脚本及使用说明 |
| `validation/验证矩阵.md` | 5 个历史实例的 vulnerable/fixed 验证矩阵 |
| `validation/验证结果模板.json` | 后续跑数据库验证时填写的结构化模板 |
| `validation/编译记录.md` | 本机 CodeQL 编译检查记录 |
| `validation/验证数据库准备记录.md` | 10 个最小 vulnerable/fixed CodeQL database 的准备与 smoke 记录 |
| `minimal-validation-databases/` | 5 个试点 CVE 的最小 source、database、日志和 SARIF 结果 |

## 当前状态

- 已整理两条候选 QL。
- 两条候选 QL 已通过本机 CodeQL 编译检查。
- 已准备 5 个试点 CVE 的 vulnerable/fixed 最小 CodeQL database，共 10 个。
- 已完成 miss/still_reports 归因，并将值身份、guard、assert、复合赋值和协议阈值建模修正回候选 QL、RuleAgent 子模板和 prompt 草案。
- 已用 CodeQL 2.25.6 和 `--rerun` 强制复测 10 个数据库：5 个 vulnerable 均在预期危险点各命中 1 条，5 个 fixed 均为 0。
- 两条规则已通过第一批机制级闭环；该结论不等同于完整 FreeBSD 历史源码上的召回率或误报率。

## 脚本使用

最终项目环境以 Linux 为准。在任意机器下载本目录后，进入 `first-ql-validation-package` 目录运行：

```bash
chmod +x scripts/run_smoke_validation.sh
./scripts/run_smoke_validation.sh --all
```

脚本默认从 `PATH` 查找 `codeql`，也可用 `CODEQL=/path/to/codeql` 指定。更多用法见 `scripts\脚本使用说明.md`。

## 本轮不做

- 不做 FreeBSD-CURRENT 大规模扫描。
- 不处理多 CVE composite diff。
- 不把 `PASS_WEAK` pattern 纳入本包。
- 不把未验证 `.ql` 当作正式规则库。
