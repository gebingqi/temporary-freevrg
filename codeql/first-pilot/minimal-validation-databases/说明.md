# 最小验证数据库说明

## 用途

本目录用于为 `first-ql-validation-package` 准备 5 个试点 CVE 的 vulnerable/fixed 最小 CodeQL C/C++ database。每个 CVE 各有一份 vulnerable harness 和一份 fixed harness，共 10 个 source root，对应生成 10 个 CodeQL database。

这些数据库只用于验证两条候选 QL 的机制行为：漏洞版是否能命中关键危险点，修复版是否因同函数前置 guard 或 `assert` 不再命中同一位置。它们不等同于完整 FreeBSD 历史源码级验证库，也不能单独作为真实版本召回率或误报率结论。

## 目录

| 目录 | 内容 |
|------|------|
| `source` | 10 个最小 C harness source root |
| `db` | 10 个 CodeQL database 输出目录 |
| `logs` | database create 日志 |
| `results` | smoke analyze SARIF 结果 |

## 建库状态

- 10 份 harness 已通过 clang `-std=c11 -c` 编译检查。
- 10 个 CodeQL database 均已创建，且每个 database 均存在 `codeql-database.yml`。
- 建库日志 `create-*.log` 均为 `STATUS=OK`，未发现 fatal/catastrophic extraction error。
- 10 次 `codeql database analyze` smoke test 均为 `STATUS=OK`。
- CodeQL C/C++ 建库对中文 source/db 路径不稳定，本轮实际使用 `D:\freevrg-codeql-work` ASCII staging 路径完成 `database create`，再将已 finalize 的数据库复制回本目录。
- CVE-2021-29631 harness 使用真实 `<assert.h>`；10 个数据库已在该表达下重新创建。
- 最终 `--rerun` 复测结果为 5 个 vulnerable 各命中 1 条、5 个 fixed 均为 0。

## 样本对应

| CVE | SA | 函数 | harness 重点 |
|-----|----|------|--------------|
| CVE-2018-17161 | FreeBSD-SA-18:15 | `handle_request` | `bp->bp_htype` 控制数组下标，fixed 加 `bp->bp_htype >= hwinfocnt` |
| CVE-2019-5604 | FreeBSD-SA-19:16 | `pci_xhci_device_doorbell` | `epid` 控制 `dev->eps[epid]`，fixed 加 `epid == 0 || epid >= XHCI_MAX_ENDPOINTS` |
| CVE-2021-29631 | FreeBSD-SA-21:13 | `pci_vt9p_notify` | `vq_getchain` 返回 `n` 控制 `iov[]` 使用，fixed 加 `assert(n >= 1 && n <= VT9P_MAX_IOV)` |
| CVE-2020-7461 | FreeBSD-SA-20:26 | `find_search_domain_name_len` | `pointed_len` 参与长度累加，fixed 加 `pointed_len < 0` |
| CVE-2021-29629 | FreeBSD-SA-21:12 | `is_valid_response` | `len` 控制 RADIUS 属性解析循环，fixed 加 `len < POS_ATTRS || len > h->in_len` |

## 限制

- harness 只保留目标函数、危险使用点、修复 guard 和必要 stub。
- harness 不复制完整 FreeBSD 源码树，也不覆盖真实构建系统、平台宏和完整数据流上下文。
- 当前结果只确认两条候选 QL 在最小机制库上的配对行为；完整历史源码召回率和误报率仍需真实版本数据库评估。
