# 聚类结果二次修订版说明

## 这是什么

`聚类结果-二次修订版` 是在 `聚类结果-修订版` 基础上，按照老师对 commit 粒度的意见更新后的 PatternAgent 输入数据包。

本目录仍然不是最终 CodeQL 规则结果，而是后续 Rule Agent 生成规则、CodeQL 编译、历史召回和 fixed-version 误报验证的输入。

本次二次修订保留原有 33 个 train≤2021 单 CVE 实例、5 个类层 pattern 和 2 个第一批 Rule Agent 试点，同时补充 patch 证据粒度字段，并把多 CVE 样本从“暂缓使用”改成“待分流集合”。

## 核心策略

语料单元仍然是 CVE，而不是 SA。

但 CVE 作为语料单元不等于必须人为拆出一个独立 commit。后续处理按证据粒度分三档：

```text
single_commit
  一 CVE 一 commit，直接作为 exact_patch 输入。

child_commit
  FreeBSD 合并提交下能找到 upstream/child commit，按子 commit 作为精确 patch 输入。

composite_commit
  确实找不到子 commit 时，保留综合 commit diff。
  每个 CVE 使用独立 CVE 描述，引导模型定位相关文件、函数、hunk、source、sink、sanitizer。
  不伪造独立 commit。
```

也就是说，多 CVE 记录可以共享同一份综合 diff 作为证据来源，但不能把综合 diff 无区分地当作每个 CVE 的干净单 CVE patch。每个 CVE 必须有自己的描述依据、定位证据和置信度标记。

## data/instances

`data/instances` 中保留 33 个 train≤2021 单 CVE 实例文件。每个实例仍记录：

- `vuln_class`：该 CVE 所属漏洞类
- `source`：攻击者可控输入来源
- `sink`：危险使用点
- `added_guard`：修复中新增或强化的关键检查
- `evidence`：新增代码行、sink 行和依据来源
- `generalizable`：是否适合作为泛化 pattern 的支撑实例

本次新增 patch 证据粒度字段：

```yaml
source_sa: FreeBSD-SA-YY:NN
fix_commits: ["..."]
patch_granularity: single_commit
evidence_mode: exact_patch
requires_review: false
```

当前 33 个实例均为 `single_commit + exact_patch`，因此可以沿用严格 grounding 检查。

5. 未通过 grounding 前，不进入 Rule Agent。

## grounding 检查

检查脚本位于：

```text
scripts/grounding_check.py
```

它检查：

- 33 个实例的 `source / sink / added_guard` 是否能对应真实样本代码
- 5 个类层 pattern 是否覆盖成员 CVE
- 试点 pattern 是否保持 `PASS_STRICT`
- 新增 patch 证据粒度字段是否存在且合法

对 `single_commit / exact_patch`：

- 沿用严格检查
- `added_guard` 优先要求命中 diff `+` 行
- sink 应能在 `after_code` 中命中

对未来 `composite_commit / cve_guided_composite`：

- 必须有单个 CVE 描述依据
- 必须有相关文件、函数或 hunk 定位
- 必须有 sink anchor 和 sanitizer/evidence anchor
- 默认最高为 `PASS_WEAK`
- 只有人工审计后标记 `requires_review:false` 且证据完整，才允许升级为严格可用输入

## 后续使用建议

第一批小范围跑通仍然按以下顺序：

1. 读取 `data/patterns/_rule_candidates.json`
2. 只选择其中两个 `PASS_STRICT` pattern
3. 基于这两个 pattern 生成 CodeQL 规则
4. 做 CodeQL 编译
5. 做历史漏洞召回
6. 做 fixed-version 误报验证

多 CVE 待分流集合暂不直接喂给 Rule Agent。它们的下一步任务是查找 child commit；若找不到，再进入 CVE-guided 综合 diff 提取。

一句话总结：

本版不再把“拆出独立 commit”作为硬要求，而是把“每个 CVE 是否有可 grounding 的 source/sink/sanitizer 证据”作为进入 Rule Agent 的判断标准。
