# RuleAgent 试点包说明

## 交付目的

本目录用于承接 `聚类结果-二次修订版` 中两个 `PASS_STRICT` class pattern，整理成后续 RuleAgent 可直接读取、可审计、可验证的输入包。

本包以 RuleAgent 输入准备与模板约束为主。

当前仓库版本额外保留了两条工作草案 `.ql`，用于对照输入与后续验证包；但这里仍不保留 CodeQL 编译日志、`.sarif` 结果或 vulnerable/fixed 版本扫描输出。

## 试点范围

第一批只包含两个 pattern：

1. `missing-array-bounds-check-on-network-controlled-index`
2. `missing-minimum-length-check-on-network-protocol-packet`

准入条件：

- `patch_granularity = single_commit`
- `evidence_mode = exact_patch`
- `requires_review = false`
- grounding 结果为 `PASS_STRICT`
- `rule_candidate = pilot`

明确排除：

- `PASS_WEAK` pattern
- `rule_candidate: defer`
- 未完成 child/upstream commit 查找的多 CVE 综合 diff
- 未通过人工复核的 `cve_guided_composite` 产物

## 使用顺序

1. 先读 `RuleAgent输入输出规范.md`。
2. 按 pattern 类型读取 `子模板` 中对应说明。
3. 读取具体试点目录下的 `rule_input.json`。
4. 由 RuleAgent 生成候选 `.ql`。
5. 按 `验证计划.md` 做 CodeQL 编译、历史召回和 fixed-version 误报验证。
6. 验证失败时优先修正 pattern、RuleAgent prompt 或子模板，不先扩大样本。
