# 数组描述符上界检查子模板

## 适用 pattern

```text
missing-array-bounds-check-on-network-controlled-index
```

## 漏洞机制

外部可控整数字段被用作数组下标、数组访问数量、iovec 数组长度或 VirtIO 描述符链长度。使用前缺少与数组容量、协议上限或有效区间的比较，导致越界读取或写入。

抽象路径：

```text
external integer -> index/count -> array or descriptor use -> missing upper-bound guard
```

## Source 建模建议

优先识别以下来源：

- 网络请求结构体字段，例如 BOOTP 包中的硬件类型字段。
- VM 访客通过 MMIO、寄存器、doorbell 写入的 endpoint/index 字段。
- VirtQueue 或描述符环返回的链长度，例如 `vq_getchain` 返回值。

RuleAgent 可先以局部变量来源和函数调用返回值为起点，不必第一版完成跨函数 taint。

## Sink 建模建议

优先识别以下使用点：

- `array[index]`
- 以外部整数控制的 `struct iovec iov[MAX]` 使用数量
- 描述符数组或 endpoint 数组访问
- 以 `n`、`idx`、`epid` 等外部整数控制的批量填充或读取

## Sanitizer 建模建议

使用点前同一函数内出现以下检查，可视为有效 guard：

- `idx >= capacity`
- `idx >= MAX`
- `idx == 0 || idx >= MAX`
- `n > MAX`
- `n >= 1 && n <= MAX`
- `assert(n >= 1 && n <= MAX)`

## 负例

- 索引来自内部循环计数，且循环边界直接由数组长度控制。
- 索引来自固定枚举值或编译期常量。
- 使用点前已有等价 min/max 范围检查。
- 检查只验证非空、日志状态或错误码，不限制索引范围。

## QL 形态建议

第一版可以优先实现函数内结构模式：

```ql
from Function f, Expr idx, Expr use, Stmt useSite
where
  isExternalIntegerLike(idx, f) and
  isArrayOrDescriptorUse(use, idx, useSite) and
  not hasUpperBoundGuard(idx, useSite, f)
select useSite, "Externally controlled index or descriptor count is used without an upper-bound check."
```

## few-shot anchors

| CVE | source | sink | sanitizer |
|-----|--------|------|-----------|
| CVE-2018-17161 | `bp->bp_htype` | `hwaddr[bp->bp_htype]` / `hwinfocnt` | `if (bp->bp_htype >= hwinfocnt)` |
| CVE-2019-5604 | `epid` from XHCI doorbell | `dev->eps[epid]` | `if (epid == 0 || epid >= XHCI_MAX_ENDPOINTS)` |
| CVE-2021-29631 | `n = vq_getchain(...)` | `iov[VT9P_MAX_IOV]` usage | `assert(n >= 1 && n <= VT9P_MAX_IOV)` |

## 验证重点

- 如果 vulnerable 版本 miss，优先检查 source 是否过窄，特别是 `vq_getchain` 返回值。
- 如果 fixed 版本仍报，优先补充 `assert` 和双条件 guard 的识别。
- 如果误报多，限制 sink 必须是数组/描述符实际访问，而不是所有整数比较。
