# missing-minimum-length-check-on-network-protocol-packet

## 试点定位

该 pattern 用于检测网络协议解析中从报文解码出的长度、偏移或属性长度字段只做上界检查，缺少非负性或协议最小长度检查的问题。

本 pattern 是第一批 RuleAgent 试点，状态为：

```text
single_commit / exact_patch / PASS_STRICT
```

## 类层摘要

- source：网络报文中的 Length 字段、TLV 属性长度或压缩域名指针偏移。
- sink：解码长度控制循环边界、指针推进、缓冲区读取范围或解析位置推进。
- sanitizer：`len < MIN_PROTOCOL_LENGTH`、`pointed_len < 0`、`len < MIN || len > buffer_len` 等非负或双侧检查。
- severity：medium
- confidence：high
- generalizable：yes
- rule_type：intra-procedural
- module_scope：cross-module

## 历史实例

| CVE | SA | 子系统 | 文件 | 函数/位置 | Source | Sink | Sanitizer |

## RuleAgent 检测意图

RuleAgent 应寻找：

```text
网络包长度/偏移 -> 循环或指针推进/属性读取 -> 缺少最小长度或非负检查
```

优先报告循环、指针推进、属性读取或解析位置更新位置，不报告 source。

## 负例

- 长度字段来自已验证的固定头部，调用前已有统一协议解码器保证最小长度。
- 解析循环每次推进前都检查剩余缓冲区至少包含 TLV 头部和声明长度。
- 长度只用于日志输出或错误消息，不影响读取、循环或指针推进。
