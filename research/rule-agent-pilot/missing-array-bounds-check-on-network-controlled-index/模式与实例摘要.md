# missing-array-bounds-check-on-network-controlled-index

## 试点定位

该 pattern 用于检测外部可控整数被用作数组索引、数组访问数量或 iovec/描述符数组数量时，缺少上界或有效区间检查的问题。

本 pattern 是第一批 RuleAgent 试点，状态为：

```text
single_commit / exact_patch / PASS_STRICT
```

## 类层摘要

- source：网络请求或 VM 访客写入的整数字段，包括 BOOTP 请求字段、PCI MMIO 寄存器、VirtIO 描述符环。
- sink：外部整数作为数组索引访问宿主侧数组，或作为 iovec/描述符数组访问数量。
- sanitizer：数组访问前检查 `index >= capacity`、`index == 0` 等无效范围，或描述符访问前断言 `n >= 1 && n <= MAX_IOV`。
- severity：high
- confidence：high
- generalizable：yes
- rule_type：intra-procedural
- module_scope：cross-module

## 历史实例

| CVE | SA | 子系统 | 文件 | 函数/位置 | Source | Sink | Sanitizer |
|-----|----|--------|------|-----------|--------|------|-----------|
| CVE-2018-17161 | FreeBSD-SA-18:15 | bootpd | `libexec/bootpd/bootpd.c` | `handle_request` | BOOTP 请求包中的 `bp_htype` | `hwaddr[bp->bp_htype]` / `hwinfocnt` | `if (bp->bp_htype >= hwinfocnt)` |
| CVE-2019-5604 | FreeBSD-SA-19:16 | bhyve | `usr.sbin/bhyve/pci_xhci.c` | `pci_xhci_device_doorbell` | VM 访客写入 XHCI Doorbell 的 `epid` | `dev->eps[epid]` | `if (epid == 0 || epid >= XHCI_MAX_ENDPOINTS)` |
| CVE-2021-29631 | FreeBSD-SA-21:13 | bhyve | `usr.sbin/bhyve/pci_virtio_9p.c` | `pci_vt9p_notify` | VM 访客通过 VirtIO 描述符环传入的 iovec 链长度 | `vq_getchain` 返回值控制 `iov[]` 使用 | `assert(n >= 1 && n <= VT9P_MAX_IOV)` |

## RuleAgent 检测意图

RuleAgent 应寻找：

```text
外部整数 -> 数组索引/描述符数量 -> 缺少上界或有效区间检查
```

优先报告数组访问、描述符数组访问或 iovec 使用点，不报告 source。

## 负例

- 索引来自内部循环计数，且循环上界由数组长度控制。
- 索引来自固定枚举或编译期常量。
- 数组访问前已有等价上界检查、零值拒绝或 `assert(n >= 1 && n <= MAX)`。
